Introduction
--------------
Architecture
~~~~~~~~~~~~~~
The SPI driver follows Linux framework: :file:`spi-dev.c`, :file:`spi-mem.c`, :file:`spi.c`. The SPI software architecture is illustrated as below.

.. figure:: figures/spi_software_arch_linux.svg
   :scale: 140%
   :align: center

   SPI software architecture

For more details on Linux SPI framework, refer to https://www.kernel.org/doc/html/v5.4/spi/spi-summary.html.

DTS Configuration
~~~~~~~~~~~~~~~~~~~~
SPI DTS Property
^^^^^^^^^^^^^^^^^^^^
.. code-block::

   spi0: spi0@0x400E8000 {
      compatible = "realtek,ameba-spi";
      rtk,spi-index = <0>;
      reg = <0x400E8000 0x100>,
      <0x41000000 0x30>; // reg base for spi role setting.
      interrupts = <GIC_SPI 40 IRQ_TYPE_LEVEL_HIGH>;
      clocks = <&rcc RTK_CKE_SPI0>;
      clock-names = "rtk_spi_clk";
      spi-max-frequency = <100000000>;
      spi-deactivate-delay = <0>;
      rtk,spi-for-kernel = <0>;
      rtk,spi-slave-mode = <0>;
      rtk,spi-dma-en = <0>;
      dmas = <&dma 16>,<&dma 17>;
      dma-names = "spi0-tx","spi0-rx";
      rtk,spi-master-poll-mode = <0>;
      rtk,spi-default-cs = <0>;
      rtk,max-cs-num = <2>;
      //only master mode set SPI_CS to GPIO, so spi0_pins only set SPI0_CLK, SPI0_MOSI and SPI0_MISO.
      rtk,spi-cs-gpios = <&gpiob 17 0>;
      status = "disabled";
   };

The SPI DTS configurations of SPI are listed as below.

.. table::
   :width: 100%
   :widths: auto

   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | Property                 | Description                                                                                    | Configurable? |
   +==========================+================================================================================================+===============+
   | compatible               | The description of SPI driver: ``"realtek,ameba-spi"``.                                        | No            |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | reg                      | The hardware address and size for SPI device: ``<0x400E8000 0x100>``.                          | No            |
   |                          |                                                                                                |               |
   |                          | The hardware address for SPI role setting: ``<0x41000000 0x30>``.                              |               |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | interrupts               | The GIC number of SPI device: ``<GIC_SPI 40 IRQ_TYPE_LEVEL_HIGH>``.                            | No            |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | clocks                   | The clock of SPI device: ``<&rcc RTK_CKE_SPI0>``.                                              | No            |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | spi-max-frequency        | The maximum frequency for SPI device: ``<100000000>``.                                         | No            |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | spi-deactivate-delay     | Delay time for SPI deactivate. Default: 0.                                                     | Yes           |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | rtk,spi-for-kernel       | If spi controller is registered for kernel device, register board inifo in child-node probe.   | 0/1           |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | rtk,spi-slave-mode       | - 0: master SPI                                                                                | 0/1           |
   |                          | - 1: slave SPI                                                                                 |               |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | rtk,spi-dma-en           | - 0: CPU transfer                                                                              | 0/1           |
   |                          | - 1: DMA transfer                                                                              |               |
   |                          |                                                                                                |               |
   |                          | | If setting to 1, ``dmas = <&dma 3>`` means the physical channel allocated to SPI.            |               |
   |                          | | ``dma-names = "dma_spi"`` means the DMA channel name.                                        |               |
   |                          | | If setting to 0, dmas and dma-names are useless parameters.                                  |               |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | rtk,spi-master-poll-mode | - 0: interrupt mode                                                                            | 0/1           |
   |                          | - 1: poll mode                                                                                 |               |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | rtk,spi-default-cs       | If chip is not selected particularly, it will be the default chip. Default: 0.                 | Yes           |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | rtk,max-cs-num           | The maximum number of chip. Default: 2.                                                        | Yes           |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | rtk,spi-clk-divider      | Divide clock by this value. The lager value it is, the slower the SPI transfer will be.        | Yes           |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | rtk,spi-cs-gpios         | The gpio for SPI.                                                                              | Yes           |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+
   | status                   | Whether enable this device.                                                                    | Yes           |
   |                          |                                                                                                |               |
   |                          | - disabled                                                                                     |               |
   |                          | - endbled                                                                                      |               |
   +--------------------------+------------------------------------------------------------------------------------------------+---------------+

SPI Group
^^^^^^^^^^^^^^
The SPI group has two SPI devices named spi0 and spi1. The usage of these two SPI devices are the same.

.. code-block::

   aliases {
      spi0 = "/ocp/spi0@0x400E8000";
      spi1 = "/ocp/spi1@0x400E9000";
   };

APIs
--------
APIs for User Space
~~~~~~~~~~~~~~~~~~~~~~~
Master Mode
^^^^^^^^^^^^^^^^^^^^^^
The SPI interfaces for user space are provided by ``<linux>/drivers/spi/spi-dev.c``.

Here are some commonly used APIs to control the SPI device:

.. table::
   :width: 100%
   :widths: auto

   +----------------+--------------------------------------------------------+
   | Interfaces     | Introduction                                           |
   +================+========================================================+
   | spidev_open    | Open an SPI device.                                    |
   +----------------+--------------------------------------------------------+
   | spidev_release | Release an SPI device.                                 |
   +----------------+--------------------------------------------------------+
   | spidev_ioctl   | Configure SPI parameters, control read/write SPI data. |
   +----------------+--------------------------------------------------------+
   | spidev_write   | Call SPI driver to send a message.                     |
   +----------------+--------------------------------------------------------+
   | spidev_read    | Call SPI driver to read a message.                     |
   +----------------+--------------------------------------------------------+

.. note::
   For more details, refer to https://www.kernel.org/doc/html/v5.4/spi/spidev.html.

The SPI demo for user space locates at ``<test>/spi``.

Slave mode
^^^^^^^^^^^^^^^^^^^^
The interface for SPI slave in user space is the same to master mode. To use SPI slave mode, just change master/slave mode in DTS described in :ref:`DTS Configuration <dts_configuration>`.

The usage of SPI slave mode and SPI master mode is the same. To use SPI slave mode, let ``rtk, spi-slave-mode = 1`` in DTS described in :ref:`DTS Configuration <dts_configuration>`. Otherwise, set ``rtk, spi-slave-mode = 0`` for master mode.

APIs for Kernel Space
~~~~~~~~~~~~~~~~~~~~~~~
Master Mode
^^^^^^^^^^^^^^^^^^^^^^
The SPI interfaces for kernel space are described in ``<linux>/include/linux/spi/spi.h``.

.. table::
   :width: 100%
   :widths: auto

   +---------------------+-------------------------------------------------------------+
   | Interfaces          | Introduction                                                |
   +=====================+=============================================================+
   | spi_sync_transfer   | Synchronous SPI data transfer                               |
   +---------------------+-------------------------------------------------------------+
   | spi_write           | SPI synchronous write                                       |
   +---------------------+-------------------------------------------------------------+
   | spi_read            | SPI synchronous read                                        |
   +---------------------+-------------------------------------------------------------+
   | spi_write_then_read | This copies txbuf and rxbuf data; for small transfers only! |
   +---------------------+-------------------------------------------------------------+

For more details, refer to https://www.kernel.org/doc/html/v5.4/spi/spidev.html.

Slave Mode
^^^^^^^^^^^^^^^^^^^^
The interface for SPI slave in kernel space is the same to master mode. To use SPI slave mode, just change master/slave mode in DTS described in :ref:`DTS Configuration <dts_configuration>`.

The usage of SPI slave mode and SPI master mode is the same. To use SPI slave mode, let ``rtk, spi-slave-mode = 1`` in DTS described in :ref:`DTS Configuration <dts_configuration>`. Otherwise, set ``rtk, spi-slave-mode = 0`` for master mode.
