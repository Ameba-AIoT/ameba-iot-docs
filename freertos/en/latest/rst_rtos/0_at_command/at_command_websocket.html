WebSocket AT Commandsïƒ

The WebSocket AT Commands are not enabled by default, and you can enable WebSocket AT commands by the following selections:

  • Select CONFIG AT CMD -> ATCMD Mode -> MCU Control

  • Select CONFIG AT CMD -> Enable WEBSOCKET

AT+WSGLCFGïƒ

Descriptionïƒ

Configure global parameters for WebSocket.

Commandïƒ

AT+WSGLCFG=[<connect_timeout>,<recv_timeout>,<send_timeout>,<send_blocktime>,<keepalive_idle>,<keepalive_interval>,<keepalive_count>]

Responseïƒ

OK

Or

ERROR:<error_no>

Parameterïƒ

<connect_timeout>:

The timeout for establishing WebSocket connection in seconds.

  • A nonnegative number.

  • It is 0 as default.

<recv_timeout>:

The timeout for receiving data, value for SO_RCVTIMEO option.

  • A nonnegative number.

  • It is 0 as default.

<send_timeout>:

The timeout for sending data, value for SO_SNDTIMEO option.

  • A nonnegative number.

  • It is 0 as default.

<send_blocktime>:

The block time for sending data.

  • A nonnegative number.

  • It is 0 as default.

<keepalive_idle>:

The value for TCP_KEEPIDLE option.

  • A nonnegative number.

  • It is 0 as default.

<keepalive_interval>:

The value for TCP_KEEPINTVL option.

  • A nonnegative number.

  • It is 0 as default.

<keepalive_count>:

The value for TCP_KEEPCNT option.

  • A nonnegative number.

  • It is 0 as default.

Error Numberïƒ

1:

Parameters error.

Exampleïƒ

AT+WSGLCFG=5,2000,2000,0,3,5,3
OK

AT+WSGLCFG=0,,2000,0
OK

AT+WSCFGïƒ

Descriptionïƒ

Configure parameters for a WebSocket connection.

Commandïƒ

AT+WSCFG=<link_id>,<ping_intv_sec>,<ping_timeout_sec>[,<buffer_size>,<max_queue_size>,<protocol>,<version>,<stable_buf_num>]

Responseïƒ

OK

Or

ERROR:<error_no>

Parameterïƒ

<link_id>:

The connect id.

  • [0,2]

<ping_intv_sec>:

The send interval of ping in seconds.

  • [1,7200]

  • It is 10 as default.

<ping_timeout_sec>:

The timeout of ping in seconds.

  • [1,7200]

  • It is 120 as default.

<buffer_size>:

The buffer size of tx and rx.

  • [1,16384]

  • It is 1500 as default.

<max_queue_size>:

The max size of queue.

  • [1,50]

  • It is 3 as default.

<protocol>:

The string of subprotocols using the WebSocket protocol.

  • It is â€œchat, superchatâ€ as default.

<version>:

The string of version using the WebSocket protocol.

  • It is â€œ13â€ as default.

<stable_buf_num>:

The stable buffer number in send queue.

  • A positive number and can not be greater than <max_queue_size>.

  • It is 3 as default.

Error Numberïƒ

1:

Parameters error.

2:

Memory error.

Exampleïƒ

AT+WSCFG=0,5,100,2000
OK

AT+WSHEADïƒ

Descriptionïƒ

Configure request headers of WebSocket. Only one WebSocket request header can be set at a time, but it can be set multiple times.

Commandïƒ

AT+WSHEAD=<link_id>,<req_header_len>[,<req_header>]

Responseïƒ

OK

Or

ERROR:<error_no>

Parameterïƒ

<link_id>:

The connect id.

  • [0,2]

<req_header_len>:

The length of request header.

  • If <req_header_len> is 0, it will clear all WebSocket request headers previously set.

<req_header>:

The string of request header.

  • The format is key: value.

Error Numberïƒ

1:

Parameters error.

2:

Memory error.

Exampleïƒ

AT+WSHEAD=0,26,Origin: http://example.com

OK

AT+WSHEADRAWïƒ

Descriptionïƒ

Configure request headers of WebSocket with TT mode. Only one WebSocket request header can be set at a time, but it can be set multiple times.

Commandïƒ

AT+WSHEADRAW=<link_id>,<req_header_len>

Responseïƒ

OK

Or

ERROR:<error_no>

Parameterïƒ

<link_id>:

The connect id.

  • [0,2]

<req_header_len>:

The length of request header.

  • If <req_header_len> is 0, it will clear all WebSocket request headers previously set.

Error Numberïƒ

1:

Parameters error.

2:

Memory error.

3:

TT mode error.

Exampleïƒ

AT+WSHEADRAW=0,26

enter tt mode

// When the following printing occurs on MCU, you can input request header.
>>>
// Input request header, then press ENTER.
Origin: http://example.com

exit tt mode

OK

Note

The total number of request headers that can be set for WSHEAD and WSHEADRAW cannot exceed 10.

AT+WSOPENïƒ

Descriptionïƒ

Open a WebSocket connection.

Commandïƒ

AT+WSOPEN=<link_id>,<host>[,<path>],<conn_type>[,<certificate_index>]

Responseïƒ

OK

Or

ERROR:<error_no>

Parameterïƒ

<link_id>:

The connect id.

  • [0,2]

<host>:

The string of host name.

<path>:

The string of resource path.

<conn_type>:

The type of connection.

  • 1:TCP.

  • 2:SSL. Do not verify certificate.

  • 3:SSL. Verify server certificate.

  • 4:SSL. Provide client certificate.

  • 5:SSL. Verify server certificate and provide client certificate.

<certificate_index>:

Specify which set of certificate suite to use in the file system.

Error Numberïƒ

1:

Parameters error.

2:

Memory error.

3:

Connect error.

4:

Ssl certificate error.

Exampleïƒ

AT+WSOPEN=0,websocket-echo.com,,1

// Here, display connect result.
>>>>>> Connect to websocket server success
OK

AT+WSSENDïƒ

Descriptionïƒ

Send data to a webSocket connection.

Commandïƒ

AT+WSSEND=<link_id>,<length>[,<use_mask>,<opcode>],<data>

Responseïƒ

OK

Or

ERROR:<error_no>

Parameterïƒ

<link_id>:

The connect id.

  • [0,2]

<length>:

The length of data.

  • A positive number and can not be greater than <buffer_size>.

<use_mask>:

Whether to use mask.

  • 0: Do not use mask.

  • 1: Use mask(default).

<opcode>:

The type of the data.

  • 0x0. Continuation.

  • 0x1. Text(default).

  • 0x2. Binary.

  • [0x3,0x7]. Reserve.

  • 0x8. Close.

  • 0x9. Ping.

  • 0xA. Pong.

  • [0xB,0xF]. Reserve.

<data>:

The string of data.

Error Numberïƒ

1:

Parameters error.

Exampleïƒ

AT+WSSEND=0,5,1,1,hello
OK
