MQTT AT Commandsïƒ

The MQTT AT commands supports all the AT commands mentioned on this page. The MQTT AT commands is disabled by default. You can enable MQTT AT commands by the following steps:

  • Navigate to your target project and open configuration menu.

  • Select CONFIG AT CMD -> Enable MQTT

The default ATCMD mode is LOGUART. You can enable MCU control mode by the following selections:

  • Select CONFIG AT CMD -> ATCMD Mode -> MCU Control

  • Save and Exit.

Note

The TT mode is only supported for MCU control mode. If you need use TT mode for AT commands, you need enable MCU control mode. The details of TT mode refer to Transparent Transmission.

AT+MQTTCFGïƒ

Descriptionïƒ

Set MQTT connection configuration.

Note

It is optional. If user donâ€™t set connection configuration with this command, the connection configuration will set as default value.

Commandïƒ

AT+MQTTCFG=<link_id>,<keepalive>,<timeout>,<disable_clean_session>[,<lwt_topic>,<lwt_msg>,<lwt_qos>,<lwt_retain>]

Responseïƒ

OK

Or

ERROR:<error_no>

Parameterïƒ

<link_id>:

Link ID.

  • Range: [0,3]

<keepalive>:

Timeout of MQTT ping.

  • Unit: second. Default: 60s.

<timeout>:

Timeout of MQTT send/receive.

  • Unit: millisecond. Default: 60000ms.

<disable_clean_session>:

Set MQTT clean session.

  • 0: enable clean session.

  • 1: disable clean session.

<lwt_topic>:

LWT (Last Will and Testament) message topic.

<lwt_msg>:

LWT message.

<lwt_qos>:

LWT QoS.

  • Which can be set to 0, 1, or 2. Default: 0

<lwt_retain>:

LWT retain.

  • Which can be set to 0 or 1. Default: 0.

Error Numberïƒ

1:

Parameters error.

2:

Malloc error.

8:

MQTT client already connected to broker.

Exampleïƒ

For the case set with LWT:

AT+MQTTCFG=0,20,1000,0,topic/lwt,This is a lwt message,0,0
OK

For the case set without LWT:

AT+MQTTCFG=0,20,1000,0
OK

AT+MQTTCONNïƒ

Descriptionïƒ

Connect to MQTT broker.

Commandïƒ

AT+MQTTCONN=<link_id>,<host>,[<port>],<client_id>,<conn_type>[,<cert_index>,<username>,<password>]

Responseïƒ

OK

When AT receives ACK from host, it will prompt:

$ACK

Or

ERROR:<error_no>

Parameterïƒ

<link_id>:

Link ID.

  • Range: [0,3]

<host>:

MQTT broker domain.

<port>:

MQTT broker port.

  • Range: [0,65535]

  • If donâ€™t set or set to 0, it will be set as default. Default: 1883 for MQTT over TCP, 8883 for MQTT over TLS.

<client_id>:

MQTT client ID.

<conn_type>:

MQTT connection type.

  • 1: MQTT over TCP.

  • 2: MQTT over TLS (no certificate verify).

  • 3: MQTT over TLS (verify server certificate).

  • 4: MQTT over TLS (provide client certificate).

  • 5: MQTT over TLS (verify server certificate and provide client certificate).

<cert_index>:

Certificate index. Default 0.

<username>:

The username to login to the MQTT broker.

<password>:

The password to login to the MQTT broker.

Error Numberïƒ

1:

Parameters error.

2:

Malloc error.

3:

Connection error.

8:

MQTT client already connected to broker.

14:

MQTT main task create error.

Exampleïƒ

For the case set with cert_index, <username> and <password>:

AT+MQTTCONN=0,broker.hivemq.com,1883,publisher,1,0,rs,12345678
OK
$ACK

For the case set without cert_index, <username> and <password>:

AT+MQTTCONN=0,broker.hivemq.com,1883,publisher,1
OK
$ACK

For the case set without port:

AT+MQTTCONN=0,broker.hivemq.com,,publisher,1
OK
$ACK

AT+MQTTSUBïƒ

Descriptionïƒ

Subscribe a topic from MQTT broker. Up to 5 topics can be subscribed.

Commandïƒ

AT+MQTTSUB=<link_id>,<topic>,<qos>

Responseïƒ

OK

If the topic has been subscribed before, it will prompt:

ALREADY SUBCRIBE

When AT receives ACK from host, it will prompt:

$ACK

Or

ERROR:<error_no>

When AT receives MQTT messages of the subscribed topic, it will prompt:

$+MQTTSUBRECV:<link_id>,<msg_id>,<topic>,<msg>,<msg_len>,<qos>,<dup>,<retain>

Parameterïƒ

<link_id>:

Link ID.

  • Range: [0,3]

<topic>:

The string of topic to be subscibed.

<qos>:

The QoS value.

  • Which can be set to 0, 1, or 2. Default: 2

Error Numberïƒ

1:

Parameters error.

2:

Malloc error.

7:

Not connect to the host.

10:

Can not subscribe this topic.

13:

Time out.

Exampleïƒ

AT+MQTTSUB=0,LASS/Test/Pm25Ameba/#,1
OK
$ACK

When AT receives MQTT messages of the subscribed topic:

$+MQTTSUBRECV:0,401,LASS/Test/Pm25Ameba/FT1_018,hello from AMEBA 43,19,1,0,0

AT+MQTTPUBRAWïƒ

Descriptionïƒ

Publish a message for specific topic of specified length.

Commandïƒ

AT+MQTTPUBRAW=<link_id>,<msg_id>,<topic>,<length>,<qos>,<retain>

Responseïƒ

> > >

The symbol >>> indicates that AT service is ready for receiving serial data, and you can enter the data now. When the requirement of message length determined by the parameter <length> is met, the transmission starts. If the transmission is successful, it returns:

OK

When AT receives ACK from host, it will prompt:

$ACK

Or

ERROR:<error_no>

Parameterïƒ

<link_id>:

Link ID.

  • Range: [0,3]

<msg_id>:

MQTT message id.

  • Range: [0,65535]

<topic>:

MQTT topic.

  • It is suggested to fill a topic subscribed before.

<length>:

Length of MQTT message.

<qos>:

QoS of message.

  • Which can be set to 0, 1, or 2. Default: 0.

<retain>:

Retain value.

  • Which can be set to 0 or 1. Default: 0.

Error Numberïƒ

1:

Parameters error.

2:

Malloc error.

5:

TT mode start failed.

7:

Not connect to the host.

9:

Publish failed.

13:

Time out.

Exampleïƒ

AT+MQTTPUBRAW=0,2,LASS/Test/Pm25Ameba/FT1_018,14,1,1
>>>
This is a test
OK
$ACK
