Overviewïƒ

The CHIP_EN is a reset control module designed to manage system-level reset logic.

CHIP_EN supports three reset modes: Level Reset (default), Interrupt Reset, and Pulse Reset. The module offers debounce function, allowing user-configurable debounce time between 0Î¼s and 16ms, with a default setting of 100Î¼s.

User can control the input signal to the CHIP_EN module via an external button or a host device.

The initial power-on sequence should follow the specification in the To be Confirmed: CHIP_EN section of the datasheet.

Note

  • The moduleâ€™s timer clock has an approximately 50% deviation, which must be considered when setting time-related thresholds.

  • The duration of input signal mentioned below refers to the time post-debounce processing.

Operation Modesïƒ

CHIP_EN supports three operation modes for distinct application scenarios:

  • Level Reset Mode is suitable for scenarios requiring basic reset functionality.

  • When the device needs to enter low power mode or requires wake-up function, Interrupt Reset Mode can be used. This functions is similar to a smartphoneâ€™s power button:

    • Short press: Toggles screen on/off.

    • Long press: Prompts the user to reboot the device or not.

  • For scenarios that a button pressed requires a reset and different responses are needed based on the duration of the press, Pulse Reset Mode can be used. Similar to that router device is restored to factory settings.

Level Reset Modeïƒ

In this mode, the system stays powered off when the CHIP_EN module receives a low-level input signal. Once this signal switches to a high-level, the SoC initiates a reboot sequence.

Interrupt Reset Modeïƒ

Interrupt Reset Mode is primarily used for power-saving scenarios.

This mode supports two interrupt events: Short-Press interrupt and Long-Press interrupt, as well as a Reset event.

Short-Press Interrupt Event:

This event is triggered when the low-level duration (Tlow) exceeds the user-defined short-press threshold (Tsp).

Long-Press Interrupt Event:

This event is triggered when the low-level duration (Tlow) exceeds the sum of the short-press threshold (Tsp) and the long -ress threshold (Tlp).

Reset Event:

When a long-press interrupt event triggered, if the program doesnâ€™t clear the long-press interrupt status within Tack, a reset will occur, as shown by the Reset in Reset occurs when no-ack after a long-press interrupt .

The SoC will reboot when the signal goes to a high-level.

Attention

  • After rebooting, the CHIP_EN will enter Level Reset Mode. To operate in Interrupt Reset Mode, the working mode needs to be reconfigured.

Pulse Reset Modeïƒ

In this mode, a low-level input signal to CHIP_EN directly triggers a reset operation and reboot the SoC.

After rebooting, users can continuously monitor the low-level input signal via API CHIPEN_IsPress(), to distinguish between short press and long press operations.

Attention

  • Once the chip is configured in pulse reset mode, the software cannot change this mode anymore. Only powering off the chip will revert it to Level Reset Mode.

For Pulse Reset Mode, when a low-level signal is detected on the CHIP_EN input, the SoC will be reboot immediately.
