解决方案

如图所示，Ameba通过UART/SPI等接口和主机MCU连接，主机MCU通过AT指令控制网络设备和传输网络数据。

如下图所示, 在WHC FullMAC 方案中, Ameba作为网卡, 通过UART/SPI/SDIO/USB与主机MCU连接, 为主机MCU提供网络连接功能.

主机MCU负责运行TCPIP协议栈, 用户可以在主机MCU通过scoket接口开发应用程序.

  • 主机为linux: 用户可以使用标准的wpa_supplicant 和 标准的 linux WiFi API进行产品开发.

    • 支持P2P 和 NAN

  • 主机为rtos: 用户可以使用标准的realtek Wi-Fi APIs进行产品开发.

  • WHC FullMAC DoC

如下图所示, 在WHC Bridge 方案中, Ameba作为网卡, 通过UART/SPI/SDIO/USB与主机MCU连接, 为主机MCU提供网络连接功能.

主机MCU和从机都会运行TCPIP协议栈, 用户可以在主机MCU通过scoket接口开发应用程序。

与 FullMAC 方案相比：

  • WHC Bridge 方案中, SoC中运行了TCPIP协议栈, 网络的保活不再依赖于主机MCU, 主机不需要再为网路保活而被频繁唤醒，可以为整体方案节省更多的电能, 特别适合用于IPcam等电池设备。

  • 主机不管是Linux还是RTOS，都需要使用Realtek Wi-Fi API进行Wi-Fi的开发

如下图所示, Wi-Fi R-Mesh 是一个树形Mesh网络, 用于增加Wi-Fi覆盖范围, 让距离AP比较远的设备也能获得稳定的网络连线。

Wi-Fi R-Mesh 具有以下突出优势:

  • 应用层软件开发无感, 所有的Mesh协议都在Wi-Fi驱动层实现

  • 配对组网和路径切换迅速, 为百微妙级别

  • 经过数跳的设备也具备较高的吞吐量

  • 整个Mesh网络具有很高的稳定性
