Socket AT命令集

备注

Socket AT命令集默认未启用。

可以参考 配置 SDK (menuconfig) 进入 CONFIG AT CMD 配置，勾选 Enable Socket 来启用 Socket AT命令集

AT+SKTSERVER

功能说明

配置TCP/UDP/SSL服务器监听服务。

命令格式

AT+SKTSERVER=<mode>,<loc_port>

响应格式

成功响应：

OK
+SKTSERVER:con_id=<con_id>

错误响应：

ERROR: <error_no>

参数说明

  • <mode>：协议类型

    • 0: TCP 协议

    • 1: UDP 协议

    • 2: SSL 加密传输

  • <loc_port>：本地监听端口

    • 取值范围[1, 65535]

  • <con_id>：连接标识符（初始值为1，成功建立连接时递增分配）

    • 有效范围[1,7]，0为保留标识

错误代码

  • 1: 参数数量不符

  • 2: 无效本地端口号

  • 3: 连接标识符分配失败

  • 4: 服务端任务初始化失败

  • 5: 套接字创建失败

  • 6: 套接字参数配置异常

  • 7: 端口绑定失败

  • 8: 监听状态设置失败

  • 9: TCP服务实例已存在

  • 10: 客户端接入异常

  • 11: 数据通道标识符分配失败

  • 12: UDP服务实例已存在

  • 13: 透传模式冲突

  • 14: 不支持的协议类型（SSL功能不可用）

  • 15: SSL服务绑定异常

  • 16: 证书存储空间不足

  • 17: 密钥存储空间不足

  • 18: 证书解析异常

  • 19: CA证书链解析失败

  • 20: 密钥解析异常

  • 21: SSL等待队列异常

  • 22: SSL接入失败

  • 23: SSL通道内存分配失败

  • 24: SSL上下文初始化失败

  • 25: 证书加载失败

  • 26: SSL握手失败

  • 27: SSL通道创建失败

应用示例

建立TCP监听端口1234：

AT+SKTSERVER=0,1234
OK
+SKTSERVER:con_id=1

配置UDP监听端口4321：

AT+SKTSERVER=1,4321
OK
+SKTSERVER:con_id=2

AT+SKTCLIENT

功能说明

建立TCP/UDP/SSL客户端连接。

命令格式

AT+SKTCLIENT=<mode>,<rmt_host>,<rmt_port>[,<loc_port>]

响应格式

成功响应：

OK
+SKTCLIENT:con_id=<con_id>

错误响应：

ERROR: <error_no>

参数说明

  • <mode>：协议类型

    • 0: TCP 协议

    • 1: UDP 协议

    • 2: SSL 加密传输

  • <rmt_host>：远程服务地址

    • 支持IPv4地址或域名（如192.168.1.100或www.example.com）

  • <rmt_port>：远程服务端口

    • 取值范围[1,65535]

  • <loc_port>：UDP本地绑定端口（仅UDP模式必需）

    • 取值范围[1,65535]

错误代码

  • 1: 参数数量错误

  • 2: 域名解析失败

  • 3: 无效远程端口

  • 4: 连接标识创建失败

  • 5: 客户端任务初始化失败

  • 6: 无效远程地址

  • 7: 套接字创建异常

  • 8: TCP连接挂起失败

  • 9: TCP连接建立失败

  • 10: UDP连接挂起失败

  • 11: 无效本地端口

  • 12: 端口绑定失败

  • 13: 透传模式占用

  • 14: 广播设置失败

  • 15: 组播成员设置失败

  • 16: 组播接口绑定失败

  • 17: 不支持的协议类型

  • 18: SSL连接失败

  • 19: SSL上下文分配失败

  • 20: SSL初始化异常

  • 21: SSL握手异常

  • 22: SSL挂起失败

应用示例

连接TCP服务器示例：

AT+SKTCLIENT=0,192.168.0.100,6666
OK
+SKTCLIENT:con_id=1

创建UDP客户端示例：

AT+SKTCLIENT=1,192.168.0.100,6666,1234
OK
+SKTCLIENT:con_id=2

AT+SKTDEL

功能说明

关闭指定连接或全部网络连接。

命令格式

AT+SKTDEL=<con_id>

响应格式

成功响应：

OK

错误响应：

ERROR: <error_no>

参数说明

  • <con_id>：连接标识符

    • 0: 关闭所有连接

    • 1-7: 指定连接ID

错误代码

  • 1: 空参数

  • 2: 无效连接标识格式

  • 3: 连接标识不存在

应用示例

关闭所有连接：

AT+SKTDEL=0
OK

AT+SKTTT

功能说明

启用透传数据传输模式。

命令格式

AT+SKTTT=<enable>

响应格式

成功响应：

+SKTTT:OK

错误响应：

+SKTTT:ERROR:<error_no>

参数说明

  • <enable>：模式开关

    • 1: 启用透传模式

错误代码

  • 1: 参数格式错误

  • 2: 无效连接标识

  • 3: 服务器连接不支持透传

  • 4: 多连接冲突

  • 5: 模式切换失败

备注

输入”—-“可退出透传模式

AT+SKTSEND

功能说明

通过指定连接发送数据。

命令格式

AT+SKTSEND=<data_size>,<con_id>[,<dst_ip>,<dst_port>]:<data>

响应格式

成功响应：

OK

错误响应：

ERROR: <error_no>

参数说明

  • <data_size>：有效数据长度（字节）

  • <con_id>：目标连接标识符

  • <dst_ip>：目标IP地址（仅UDP服务器模式需要）

  • <dst_port>：目标端口号（仅UDP服务器模式需要）

错误代码

  • 1: 参数错误

  • 2: 缓冲区溢出

  • 3: 无效连接标识

  • 4: UDP服务配置异常

  • 5: UDP数据发送失败

  • 6: UDP客户端异常

  • 7: TCP需指定数据通道

  • 8: TCP写入失败

AT+SKTREAD

从指定连接读取缓存数据。

命令格式

AT+SKTREAD=<con_id>,<buff_size>

响应格式

成功响应：

+SKTREAD:<recv_size>,<con_id>[,<udp_clientip>,<udp_clientport>]:<data>
OK

错误响应：

ERROR: <error_no>

参数说明

  • <con_id>：连接标识符

  • <buff_size>：接收缓冲区大小

错误代码

  • 1: 参数错误

  • 2: 缓冲区设置异常

  • 3: 无效连接标识

  • 4: UDP数据接收异常

  • 5: UDP客户端接收失败

  • 6: TCP需指定数据通道

  • 7: 连接中断

  • 8: TCP读取失败

AT+SKTRECVCFG

功能说明

配置自动数据接收模式。

自动接收模式下，只要接收到数据，就会将数据进行输出，不需要执行AT+SKTREAD来专门接收

命令格式

AT+SKTRECVCFG=<enable>

响应格式

成功响应：

OK

错误响应：

ERROR: <error_no>

参数说明

  • <enable>：自动接收开关

    • 0: 禁用自动接收

    • 1: 启用自动接收

错误代码

  • 1: 参数错误

  • 2: 任务初始化失败

AT+SKTSTATE

功能说明

查询当前所有连接状态信息。

命令格式

AT+SKTSTATE

响应格式

标准响应：

con_id:<id>,<type>,<protocol>,address:<ip>,port:<port>,socket_fd:<fd>
OK

应用示例

AT+SKTSTATE
con_id:1,server,tcp,address:192.168.0.101,port:1234,socket_fd:0
con_id:4,seed,tcp,address:192.168.0.100,port:49682,socket_fd:3
con_id:2,server,udp,address:192.168.0.101,port:4321,socket_fd:1
con_id:3,server,ssl,address:192.168.0.101,port:443,socket_fd:2
con_id:5,seed,ssl,address:192.168.0.100,port:49683,socket_fd:4
OK
