通用AT命令集

AT+TEST

描述

测试 AT 命令是否已就绪

通过此 AT 命令，用户可以检查 AT 命令是否能正常工作。

命令

AT+TEST

响应

+TEST:OK

参数

无

AT+LIST

描述

此命令用于列出当前支持的所有 AT 命令

命令

AT+LIST

响应

Common AT Command:
AT+TEST
// … … followed by other common command list.
Wi-Fi AT Command:
AT+WLSOFTAP
// … … followed by other Wi-Fi command list.
TCP/IP AT Command:
AT+SKTGETERR
// … … followed by other TCP/IP command list.
BT AT command:
AT+BLECMODE
// … … followed by other BT command list.
MQTT AT command:
AT+MQTTOPEN
// … … followed by other MQTT command list.
+LIST:OK

参数

无

AT+OTACLEAR

描述

清除应用固件OTA2签名

备注

仅LOGUART模式下支持该命令。

命令

AT+OTACLEAR

响应

+OTACLEAR:OK

参数

无

AT+OTARECOVER

描述

恢复应用固件OTA2签名

备注

仅LOGUART模式下支持该命令。

命令

AT+OTARECOVER

响应

+OTARECOVER:OK

参数

无

AT+CPULOAD

描述

周期性读取CPU负载

备注

仅LOGUART模式下支持该命令。

命令

AT+CPULOAD=<mode>[,<interval>,<count>]

响应

// If mode == 0
+CPULOAD:OK
current mode: <mode> time: <interval>(s) count: <count>(-1 for infinite)
task name  state  prio  CPU%
<cpu_stat_thread>  R  10  99.87
<task_1>  r  0  0.01
<task_2>  b  9  0.01
// Other task lists if exist.
// Wait for <interval> second, the next output, at most <count> times totally.

或

// If mode == 1
+CPULOAD:OK
current mode: <mode> time: <interval>(s) count: <count>(-1 for infinite)
// Stop here if there is an infinit statistic.

或

// If mode == 2
+CPULOAD:OK
current mode: <mode> time: <interval>(s) count: <count>(-1 for infinite)

或

// If mode == 3
+CPULOAD:OK
current mode: <mode> time: <interval>(s) count: <count>(-1 for infinite)
task name  state  prio  CPU%
<cpu_stat_thread>  R  10  99.87
<task_1>  r  0  0.01
<task_2>  b  9  0.01
// Other task lists if exist.

或

+CPULOAD:ERROR:<error_no>

参数

<mode>:

读取CPU负载的模式

  • 0: 创建CPU负载统计任务

  • 1: 如果任务正在运行，则立即输出信息，并减少计数；如果是无限次统计，则立即停止

  • 2: 更新统计时间

  • 3: 从上次统计时间开始输出信息

<interval>:

间隔时间（秒）

  • [1, INT_MAX]

<count>:

统计次数，0 或 -1 表示无限次

  • [-1, INT_MAX]

错误码

1:

参数数量错误

2:

无法成功启动任务

3:

输入模式错误

示例

AT+CPULOAD=0
current mode: 0 time: 1(s) count: 4294967295 (-1 for infinite)
task name  state  prio  CPU%
cpu_stat_thread  R  10  0.00
IDLE  r  0  99.87
inic_api_host_task  b  3  0.00
LOGUART_TASK  b  5  0.09
inic_msg_q_task  b  6  0.00
Tmr Svc  b  10  0.00
// Wait 1 second
task name  state  prio  CPU%
cpu_stat_thread  R  10  0.19
IDLE  r  0  99.78
TCP_IP  b  9  0.01
LOGUART_TASK  b  5  0.00
inic_msg_q_task  b  6  0.00
Tmr Svc  b  10  0.00
inic_api_host_task  b  3  0.00
// And so on ……
AT+CPULOAD=1
current mode: 1 time: 1(s) count: 4294967295 (-1 for infinite)
+CPULOAD:OK
// Stop now.
// Then update the last statistic time.
AT+CPULOAD=2
current mode: 1 time: 1(s) count: 4294967295 (-1 for infinite)
+CPULOAD:OK
// Output information from last statistic time by now.
AT+CPULOAD=3
current mode: 1 time: 1(s) count: 4294967295 (-1 for infinite)
+CPULOAD:OK
task name  state  prio  CPU%
cpu_stat_thread  R  10  0.19
IDLE  r  0  99.72

Tmr Svc  b  10  0.00
LOGUART_TASK  b  5  0.06
inic_api_host_task  b  3  0.00
inic_msg_q_task  b  6  0.00

描述

获取每个任务的状态和当前堆信息

命令

AT+STATE

响应

Task List:
<task_1>  <property>  <current_priority>  <stack_water_mark>  <task_number>
<task_2>  <property>  <current_priority>  <stack_water_mark>  <task_number>

AT+LOG

描述

获取、设置或清除日志级别

命令

AT+LOG=<op>[,<module>,<log_level>]

响应

+LOG:OK

或

[<module>] level = <log_level>
+LOG:OK

或

+LOG:ERROR:<error_no>

参数

<op>:

操作类型

  • 0: 获取日志级别

  • 1: 设置日志级别

  • 2: 输出所有日志级别

  • 3: 清除所有日志级别

<module>:

特定模块名称

  • 特定模块名称字符串， * 表示所有模块

  • 长度不应超过9字节

<log_level>:

日志级别

  • [0,5]

错误码

1:

参数错误

2:

设置失败

示例

// Add a new module named "module1" with level 3.
AT+LOG=1,module1,3
+LOG:OK
// Get the level of "module1"
AT+LOG=0,module1
<module2> level is 5
+LOG:OK
// Add a new module named "module2" with level 5.
AT+LOG=1,module2,5
+LOG:OK
// Print all
AT+LOG=2
[module1] level = 3
[module2] level = 5
+LOG:OK
// Clear all
AT+LOG=3
+LOG:OK

AT+RREG

描述

读取通用寄存器的值

备注

仅LOGUART模式下支持该命令。

命令

AT+RREG=<address>[,<length>,<in_byte>]

响应

// The dump value
// ……
+RREG:OK

或

+RREG:ERROR:<error_no>

参数

<address>:

Flash地址

  • [<start>, 268435455]

  • <start> 表示Flash的起始地址

<length>:

Falsh的dump长度

  • 基于<address>的长度不应溢出

<in_byte>:

以字节形式dump数据 - 如果存在 B 或 b，表示以字节的形式读取数据 - 如果不存在 B 或 b，默认以字的形式读取数据

错误码

1:

参数错误

备注

  • 输入的地址必须有效，否则可能导致系统崩溃。

  • 目前，读取的数据会显示在日志窗口中。

AT+WREG

描述

将数据写入寄存器

备注

仅LOGUART模式下支持该命令。

命令

AT+WREG=<address>,<data>

响应

+WREG:OK

或

+WREG:ERROR:<error_no>

参数

<address>:

Flash地址

  • [<start>, <end>]

  • <start> 表示Flash的起始地址，<end> 表示Flash的结束地址

<data>:

待写入至此地址的数据

错误码

1:

参数错误

备注

输入的地址必须有效，否则可能导致系统崩溃。
