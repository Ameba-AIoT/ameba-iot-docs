TCP/IP AT命令集

AT+SKTSERVER

描述

作为Socket服务器启动

命令

AT+SKTSERVER=<mode>,<loc_port>

响应

+SKTSERVER:OK
+SKTSERVER:con_id=<con_id>

或

+SKTSERVER:ERROR:<error_no>

参数

<mode>:

服务器模式

  • 0：TCP

  • 1：UDP

  • 2：SSL（是否支持取决于版本）

<loc_port>:

本地端口

  • [1, 65535]

错误码

1:

参数数量错误

2:

本地端口错误

3:

创建con_id错误

4:

创建服务器任务错误

5:

创建Socket错误

6:

设置Socket选项错误

7:

绑定错误

8:

监听错误

9:

TCP服务器已存在

10:

接收错误

11:

为种子创建con_id错误

12:

UDP服务器已存在

13:

服务器无法在TT（透明传输）模式下启动

14:

未知的连接类型（不支持SSL）

15:

监听Socket绑定到bind_ip失败（针对SSL服务器）

16:

为服务器证书分配内存失败

17:

为服务器密钥分配内存失败

18:

解析服务器证书x509_crt_parse失败

19:

解析服务器CA列表x509_crt_parse失败

20:

解析服务器密钥pk_parse_key失败

21:

SSL服务器挂起节点失败

22:

SSL服务器接受错误

23:

为SSL种子分配内存失败

24:

SSL上下文初始化失败

25:

ssl_set_own_cert错误

26:

SSL握手失败（针对SSL种子）

27:

为SSL种子创建节点失败

AT+SKTCLIENT

描述

作为Socket客户端启动

命令

AT+SKTCLIENT=<mode>,<rmt_host>,<rmt_port>[,<loc_port>]

响应

+SKTCLIENT:OK
+SKTCLIENT:con_id=<con_id>

或

+SKTCLIENT:ERROR:<error_no>

参数

<mode>:

客户端模式

  • 0: TCP

  • 1: UDP

  • 2：SSL（是否支持取决于版本）

<rmt_host>:

远程主机地址

  • IP地址的十六进制数据字符串，或网址

<rmt_port>:

远程主机端口

  • [1,65535]

<loc_port>:

本地客户端端口

  • [1,65535]

  • 仅在UDP模式下有效

错误码

1:

参数数量错误

2:

找不到主机

3:

远程端口错误

4:

创建con_id错误

5:

创建任务错误

6:

远程地址错误

7:

创建Socket错误

8:

TCP客户端挂起节点错误

9:

TCP客户端连接错误

10:

UDP客户端挂起节点错误

11:

本地端口应为1~65535

12:

绑定本地端口错误

13:

透传模式下连接已存在

14:

设置Socket广播失败

15:

设置Socket组播成员关系失败

16:

设置组播接口失败

17:

未知的连接类型（不支持SSL）

18:

SSL客户端连接远程主机端口失败

19:

SSL上下文结构内存分配失败

20:

SSL上下文初始化失败

21:

SSL握手失败

22:

SSL客户端挂起节点失败

AT+SKTDEL

描述

停止Socket服务器或客户端

命令

AT+SKTDEL=<con_id>

响应

+SKTDEL:OK

或

+SKTDEL:ERROR:<error_no>

参数

<con_id>:

待删除的con_id

  • [0,9]

  • 为0时表示删除所有con_id

错误码

1:

输入参数错误

AT+SKTTT

描述

启用透传模式

命令

AT+SKTTT=<enable>

响应

+SKTTT:OK

或

+SKTTT:ERROR:<error_no>

参数

<enable>:

启用透传模式

  • 1: 启用

错误码

1:

输入参数错误

2:

尝试启用透传模式时，未找到有效连接

3:

当前连接为服务器连接，无法启用透传模式

4:

尝试启用透传模式时，发现存在多个连接

5:

启动透传任务失败

备注

一旦成功进入透传模式，可通过输入四个连续的短划线 ---- 退出透传模式，而无须使用其他AT命令。

示例

// create TCP client, single connection
AT+SKTCLIENT=0,192.168.99.101,5001
+SKTCLIENT:OK
[AT+SKTCLIENT] con_id=1
// Enable TT mode
AT+SKTTT=1
+SKTTT:OK
// Input the data for transparent transfer, then press ENTER
Hello world!
// Input "----" to exit TT mode
----
// Return to command mode now

AT+SKTSEND

描述

发送Socket消息

命令

AT+SKTSEND=<data_size>,<con_id>[,<dst_ip>,<dst_port>]:<data>

响应

+SKTSEND:OK

或

+SKTSEND:ERROR:<error_no>

参数

<data_size>:

发送数据的字节长度

  • 必须为正数

<con_id>:

客户端的con_id

  • [1,9]

<dst_ip>:

目标IP地址

  • IP地址格式

  • 仅在UDP模式下有效

<dst_port>:

目标端口号

  • [1,65535]

  • 仅在UDP模式下有效

<data>:

冒号后的数据内容

  • 为字符串，冒号后的所有输入均视为数据内容

错误码

1:

输入参数错误

2:

缓冲区大小错误

3:

未找到con_id对应的连接

4:

UDP服务器错误

5:

UDP服务器调用 sendto 函数失败

6:

UDP客户端调用 sendto 函数失败

7:

TCP服务器应发送数据至种子

8:

TCP客户端/服务器写入数据失败

示例

// query connection information
AT+SKTSTATE
con_id:1,server,tcp,address:192.168.99.143,port:5001,socket:0
con_id:2,seed,tcp,address:192.168.99.185,port:64089,socket:1
con_id:3,server,udp,address:192.168.99.143,port:5002,socket:2
con_id:4,client,tcp,address:192.168.99.185,port:5001,socket:3
con_id:5,client,udp,address:64.233.189.104,port:8080,socket:4
+SKTSTATE:OK
// send data to TCP client(Seed) (con_id 2)
AT+ SKTSEND=14,2:Hello_World!
+SKTSEND:OK,2
// send data to UDP Server via UDP client(con_id 5)
AT+ SKTSEND=14,5:Hello_China!

+ SKTSEND:OK,5

AT+SKTREAD

描述

接收Socket消息

命令

AT+SKTREAD=<con_id>,<buff_size>

响应

+SKTREAD:OK

或

+SKTREAD:ERROR:<error_no>

参数

<con_id>:

连接ID

  • [0,3]

<buff_size>:

接收数据的字节长度

  • 必须为正数

错误码

1:

输入参数错误

2:

缓冲区大小错误

3:

未找到con_id对应的连接

4:

UDP服务器调用 recvfrom 函数失败

5:

UDP客户端/种子连接调用 recvfrom 函数失败

6:

TCP服务器应从种子连接接收数据

7:

连接已断开

8:

TCP连接读取数据失败

备注

如果通过本地创建的UDP服务器接收数据，返回结果中会附加 dst_ip 和 dst_port。

示例

// query connection information
AT+SKTSTATE
con_id:1,server,tcp,address:192.168.99.143,port:5001,socket:0
con_id:2,seed,tcp,address:192.168.99.185,port:64089,socket:1
con_id:3,server,udp,address:192.168.99.143,port:5002,socket:2
con_id:4,client,tcp,address:192.168.99.185,port:5001,socket:3
con_id:5,client,udp,address:64.233.189.104,port:8080,socket:4
+SKTSTATE:OK
// receive data “12345678” via TCP seed (con_id 2)
AT+ SKTREAD=2,1500
+SKTREAD:OK,8,2:12345678
// receive data “12345678” via UDP server(con_id 3)
AT+SKTREAD=3,1500
+SKTREAD:OK,8,3,192.168.99.185,52795:12345678
// receive data “12345678” via TCP client(con_id 4)
AT+SKTREAD=4,1500
+SKTREAD:OK,8,4:12345678

AT+SKTRECVCFG

描述

配置Socket接收模式

命令

AT+SKTRECVCFG=<enable>

响应

+SKTRECVCFG:OK

或

+SKTRECVCFG:ERROR:<error_no>

参数

<enable>:

启用/禁用自动接收数据模式

  • 0: 禁用

  • 1: 启用

错误码

1:

输入参数错误

2:

启动任务失败

备注

Once the auto receive mode is enabled, any packet received on any connection will return to host automatically in the same format as AT+SKTREAD (refer to AT+SKTREAD, response of command AT+SKTREAD) in normal transmission mode. But if under transparent transmission mode, received data will return to host without any information in the head.

AT+SKTSTATE

描述

获取当前Socket状态

命令

AT+SKTSTATE

响应

con_id:<con_id>,<server/seed(TCP_client)/client>,<tcp/udp>,address:<IP_ADDRESS>,port:<PORT>,socket:<socket id>
// Any other <con_id>s if exist.
+SKTSTATE:OK

参数

无
