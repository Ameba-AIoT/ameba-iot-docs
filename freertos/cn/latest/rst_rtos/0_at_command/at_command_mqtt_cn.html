MQTT AT命令集

MQTT AT 命令集默认禁用。可通过以下步骤启用 MQTT AT 命令集：

  1. 进入目标工程目录，并打开配置菜单：

    cd amebaxxxx_gcc_project
./menuconfig.py

  2. 选择 CONFIG AT CMD -> Enable MQTT

默认的AT命令模式为LOGUART。请通过选择 CONFIG AT CMD -> ATCMD Mode -> MCU Control 启用 MCU 控制模式。

备注

  • 仅 MCU 控制模式下支持透传。若需使用透传模式，需启用 MCU 控制模式。

  • 参考 透传模式 了解相关细节。

AT+MQTTUSERCFG

描述

配置MQTT客户端参数

备注

使用MQTT AT命令之前，请连接至AP。

命令

AT+MQTTUSERCFG=<link_id>,<conn_type>,<client_id>,<username>,<password>,<certificate_index>

响应

OK

或

ERROR:<error_no>

参数

<link_id>:

连接ID

  • [0,3]

<conn_type>:

MQTT连接类型

  • 1: 基于 TCP 的 MQTT

  • 2: 基于 TLS 的 MQTT（不验证证书）

  • 3: 基于 TLS 的 MQTT（验证服务器证书）

  • 4: 基于 TLS 的 MQTT（提供客户端证书）

  • 5: 基于 TLS 的 MQTT（验证服务器证书并提供客户端证书）

<client_id>:

MQTT 客户端 ID

<username>:

登录到 MQTT 代理的用户名

<password>:

登录到 MQTT 代理的密码

<certificate_index>:

证书索引，默认值为 0。

错误码

1:

参数错误

2:

内存分配错误

6:

读取 CA/证书/私钥失败

8:

MQTT 客户端已连接至代理

示例

  • 如果设置了 <username> 和 <password>:

    AT+MQTTUSERCFG=0,1,publisher,rs,12345678,0
OK

  • 如果没有设置 <username> 和 <password>:

    AT+MQTTUSERCFG=0,1,publisher,,,0
OK

AT+MQTTCONNCFG

描述

配置 MQTT 连接的相关参数

备注

可选，如果用户没有使用该命令进行配置，则使用默认值。

命令

AT+MQTTCONNCFG=<link_id>,<keepalive>,<timeout>,<disable_clean_session>[,<lwt_topic>,<lwt_msg>,<lwt_qos>,<lwt_retain>]

响应

OK

或

ERROR:<error_no>

参数

<link_id>:

连接ID

  • [0,3]

<keepalive>:

MQTT ping 的超时时间（秒）

  • 默认值为60

<timeout>:

MQTT 发送/接收的超时时间（毫秒）

  • 默认值为60000

<disable_clean_session>:

设置 MQTT 的清除会话选项

  • 0: 启用清除会话

  • 1: 禁用清除会话

<lwt_topic>:

LWT消息的主题

<lwt_msg>:

LWT消息

<lwt_qos>:

LWT消息的QoS

  • 可设置为 0，1，2，默认值为0

<lwt_retain>:

LWT 消息的保留选项

  • 可设置为 0或1，默认值为0

错误码

1:

参数错误

2:

内存分配错误

示例

  • 如果设置了 LWT:

    AT+MQTTCONNCFG=0,20,1000,0,topic/lwt,This is a lwt message,0,0
OK

  • 如果没有设置 LWT:

    AT+MQTTCONNCFG=0,20,1000,0
OK

AT+MQTTCONN

描述

连接至MQTT代理

命令

AT+MQTTCONN=<link_id>,<host>,<port>

响应

OK

当AT模块从主机接收到ACK时，将返回：

$ACK

或

ERROR:<error_no>

参数

<link_id>:

连接ID

  • [0,3]

<host>:

MQTT代理域名

<port>:

MQTT代理端口号

  • [1,65535]

  • 默认为：1883（基于TCP），8883（基于TLS）

错误码

1:

参数错误

2:

内存分配错误

3:

连接错误

14:

MQTT 主任务创建错误

15:

未连接到 Wi-Fi

示例

AT+MQTTCONN=0,broker.hivemq.com,1883
OK
$ACK

AT+MQTTSUB

描述

从 MQTT 代理订阅主题，最多可订阅5个主题

命令

AT+MQTTSUB=<link_id>,<topic>,<qos>

响应

OK

  • 如果主题之前被订阅过，将返回：

    ALREADY SUBCRIBE

  • 当AT模块从主机接收到ACK时，将返回：

    $ACK

    或

    ERROR:<error_no>

  • 当AT模块接收到订阅主题的 MQTT 消息时，将返回：

    $+MQTTSUBRECV:<link_id>,<msg_id>,<topic>,<msg>,<msg_len>,<qos>,<dup>,<retain>

参数

<link_id>:

连接ID

  • [0,3]

<topic>:

待订阅的主题字符串

<qos>:

QoS值

  • 可设置为 0，1，2，默认值为0

错误码

1:

参数错误

2:

内存分配错误

7:

未连接主机

10:

无法订阅此主题

13:

超时

示例

AT+MQTTSUB=0,LASS/Test/Pm25Ameba/#,1
OK
$ACK

当AT模块接收到订阅主题的 MQTT 消息时：

$+MQTTSUBRECV:0,401,LASS/Test/Pm25Ameba/FT1_018,hello from AMEBA 43,19,1,0,0

AT+MQTTPUBRAW

描述

发布指定长度的消息到特定的 MQTT 主题

命令

AT+MQTTPUBRAW=<link_id>,<msg_id>,<topic>,<length>,<qos>,<retain>

响应

> > >

>>> 符号表示AT模块已准备好接收串行数据，可以输入数据。

当输入的数据长度满足参数 length指定的要求时，传输开始。如果传输成功，将返回：

OK

当AT模块从主机接收到ACK时，将返回：

$ACK

或

ERROR:<error_no>

参数

<link_id>:

连接ID

  • [0,3]

<msg_id>:

MQTT消息ID

  • [0,65535]

<topic>:

MQTT主题

  • 建议填写之前已订阅的主题

<length>:

MQTT 消息长度

<qos>:

消息的QoS

  • 可设置为 0，1，2，默认值为0

<retain>:

保留值

  • 可设置为 0或1，默认值为0

错误码

1:

参数错误

2:

内存分配错误

5:

透传模式启动失败

7:

未连接至主机

9:

发布失败

13:

超时

示例

AT+MQTTPUBRAW=0,2,LASS/Test/Pm25Ameba/FT1_018,14,1,1
>>>
This is a test
OK
$ACK

AT+MQTTRESET

描述

重置所有 MQTT 实例，并释放资源

命令

AT+MQTTRESET

响应

OK

备注

示例

AT+MQTTRESET
OK
