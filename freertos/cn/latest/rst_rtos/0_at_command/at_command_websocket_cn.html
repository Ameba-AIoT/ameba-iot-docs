WebSocket AT命令集

WebSocket AT 命令集默认未启用，请通过以下设置来启用 WebSocket AT 命令集

  • 选择 CONFIG AT CMD -> ATCMD Mode -> MCU Control

  • 选择 CONFIG AT CMD -> Enable WEBSOCKET

AT+WSGLCFG

描述

配置 WebSocket 的全局参数

命令

AT+WSGLCFG=[<connect_timeout>,<recv_timeout>,<send_timeout>,<send_blocktime>,<keepalive_idle>,<keepalive_interval>,<keepalive_count>]

响应

OK

或

ERROR:<error_no>

参数

<connect_timeout>:

建立 WebSocket 连接的超时时间（秒）

  • 非负数

  • 默认值为0

<recv_timeout>:

接收数据的超时时间，对应 SO_RCVTIMEO 的值

  • 非负数

  • 默认值为0

<send_timeout>:

发送数据的超时时间，对应 SO_SNDTIMEO 的值

  • 非负数

  • 默认值为0

<send_blocktime>:

发送数据的阻塞时间

  • 非负数

  • 默认值为0

<keepalive_idle>:

TCP_KEEPIDLE 的值

  • 非负数

  • 默认值为0

<keepalive_interval>:

TCP_KEEPINTVL 的值

  • 非负数

  • 默认值为0

<keepalive_count>:

TCP_KEEPCNT 的值

  • 非负数

  • 默认值为0

错误码

1:

参数错误

示例

AT+WSGLCFG=5,2000,2000,0,3,5,3
OK

AT+WSGLCFG=0,,2000,0
OK

AT+WSCFG

描述

配置 WebSocket 连接的参数

命令

AT+WSCFG=<link_id>,<ping_intv_sec>,<ping_timeout_sec>[,<buffer_size>,<max_queue_size>,<protocol>,<version>,<stable_buf_num>]

响应

OK

或

ERROR:<error_no>

参数

<link_id>:

连接ID

  • [0,2]

<ping_intv_sec>:

发送 ping 的间隔时间（秒）

  • [1,7200]

  • 默认值为 10

<ping_timeout_sec>:

ping 的超时时间（秒）

  • [1,7200]

  • 默认值为 120

<buffer_size>:

发送和接收的缓冲区大小

  • [1,16384]

  • 默认值为 1500

<max_queue_size>:

队列的最大大小

  • [1,50]

  • 默认值为 3

<protocol>:

使用 WebSocket 协议的子协议字符串

  • 默认值为 chat, superchat

<version>:

使用 WebSocket 协议的版本字符串

  • 默认值为 13

<stable_buf_num>:

发送队列中的稳定缓冲区数量

  • 正整数，不能大于 max_queue_size

  • 默认值为 3

错误码

1:

参数错误

2:

内存错误

示例

AT+WSCFG=0,5,100,2000
OK

AT+WSHEAD

描述

配置 WebSocket 的请求头。一次只能设置一个 WebSocket 请求头，但可以进行多次设置。

命令

AT+WSHEAD=<link_id>,<req_header_len>[,<req_header>]

响应

OK

或

ERROR:<error_no>

参数

<link_id>:

连接ID

  • [0,2]

<req_header_len>:

请求头的长度

  • 为 0，则会清除之前设置的所有 WebSocket 请求头

<req_header>:

请求头字符串

  • 格式为 key:value

错误码

1:

参数错误

2:

内存错误

示例

AT+WSHEAD=0,26,或igin: http://example.com
OK

AT+WSHEADRAW

描述

在透传模式下配置 WebSocket 的请求头。一次只能设置一个 WebSocket 请求头，但可以进行多次设置。

命令

AT+WSHEADRAW=<link_id>,<req_header_len>

响应

OK

或

ERROR:<error_no>

参数

<link_id>:

连接ID

  • [0,2]

<req_header_len>:

请求头的长度

  • 为 0，则会清除之前设置的所有 WebSocket 请求头

错误码

1:

参数错误

2:

内存错误

3:

透传模式错误

示例

AT+WSHEADRAW=0,26
enter tt mode

// When the following printing occurs on MCU, you can input request header.
>>>
// Input request header, then press ENTER.
Origin: http://example.com

exit tt mode

OK

备注

通过 WSHEAD 和 WSHEADRAW 设置的请求头的总数不能超过 10 个。

AT+WSQUERY

描述

查询全局参数和每个连接的参数

命令

AT+WSQUERY=[<link_id>]

响应

// Here, print global parameters and parameters of connection.
Global config:
    connect_timeout: <connect_timeout>
    recv_timeout: <recv_timeout>
    send_timeout: <send_timeout>
    send_blocktime: <send_blocktime>
    keepalive_idle: <keepalive_idle>
    keepalive_interval: <keepalive_interval>
    keepalive_count: <keepalive_count>

link_id: <link_id>
    status: <status>
    ping_intv_sec: <ping_intv_sec>
    ping_timeout_sec: <ping_timeout_sec>
    buffer_size: <buffer_size>
    max_queue_size: <max_queue_size>
    protocol: <protocol>
    version: <version>
    stable_buf_num: <stable_buf_num>
    header:
        <header>
OK

或

ERROR:<error_no>

参数

<link_id>:

连接ID

  • [0,2]

  • 如果省略link_id，将打印全局参数和所有连接的参数。否则，只打印 link_id连接的参数和全局参数

错误码

1:

参数错误

示例

AT+WSQUERY=0

// Here, global parameters and parameters of connection 0 will be printed.
Global config:
    connect_timeout: 0
    recv_timeout: 0
    send_timeout: 0
    send_blocktime: 0
    keepalive_idle: 0
    keepalive_interval: 0
    keepalive_count: 0

link_id: 0
    status: 3
    ping_intv_sec: 10
    ping_timeout_sec: 120
    buffer_size: 1500
    max_queue_size: 3
    protocol:
    version: 13
    stable_buf_num: 3
    header:
        或igin: http://example.com

OK

AT+WSOPEN

描述

打开一个 WebSocket 连接

命令

AT+WSOPEN=<link_id>,<host>[,<path>],<conn_type>[,<certificate_index>]

响应

OK

或

ERROR:<error_no>

参数

<link_id>:

连接ID

  • [0,2]

<host>:

主机名字符串

<path>:

资源路径字符串

<conn_type>:

连接类型

  • 1: TCP

  • 2: SSL，不验证证书

  • 3: SSL，验证服务器证书

  • 4: SSL，提供客户端证书

  • 5: SSL，验证服务器证书并提供客户端证书

<certificate_index>:

指定文件系统中使用的证书套件

错误码

1:

参数错误

2:

内存错误

3:

连接错误

4:

SSL证书错误

示例

AT+WSOPEN=0,websocket-echo.com,,1

// Here, display connect result.
>>>>>> Connect to websocket server success
OK

AT+WSSEND

描述

通过 WebSocket 连接发送数据帧

命令

AT+WSSEND=<link_id>,<length>[,<use_mask>,<opcode>],<data>

响应

OK

或

ERROR:<error_no>

参数

<link_id>:

连接ID

  • [0,2]

<length>:

数据长度

  • 正数，且不能大于buffer_size

<use_mask>:

是否使用掩码

  • 0：不使用掩码

  • 1：使用掩码（默认）

<opcode>:

数据类型

  • 0x0：继续帧

  • 0x1：文本（默认）

  • 0x2：二进制

  • [0x3,0x7]：预留

  • 0x8：关闭

  • 0x9：Ping

  • 0xA：Pong

  • [0xB,0xF]：预留

<data>:

数据字符串

错误码

1:

参数错误

示例

AT+WSSEND=0,5,1,1,hello
OK
