RAM 布局与配置

简介

本章介绍默认内存布局及其修改方法。

RAM 内存布局

芯片集成了内部SRAM和可扩展的PSRAM，其容量如下：

  • 内部SRAM: 512KB

  • 可扩展的PSRAM: 根据不同型号，容量包括 0MB、4MB、8MB 和 16MB 等。

芯片集成了内部SRAM和可扩展的PSRAM，其容量如下：

  • 内部SRAM: 512KB

  • 可扩展的PSRAM: 根据不同型号，容量包括 0MB、4MB、8MB 和 16MB 等。

芯片集成了内部SRAM和可扩展的PSRAM，其容量如下：

  • 内部SRAM: 768KB

  • 可扩展的PSRAM: 根据不同型号，容量包括 0MB、4MB 等。

SRAM0 内存布局 (前40KB)

SRAM的前40KB被预留用于存储ROM变量、BOOT代码以及安全处理等功能，这部分内存空间用户无法使用。

修改Bootloader大小

Bootloader 容量调整

如需扩大 KM4_BOOT_RAM_S ，需确保调整后的尺寸为 4KB 对齐（MPC 保护以 4KB 为单位）。

修改步骤如下：

  1. 在 menuconfig 中配置 CONFIG Link Option 选项：

    • 选择 SRAM 存储时，KM4_BOOTLOADER_RAM_S 为 24KB

    • 选择 Flash 存储时，KM4_BOOTLOADER_RAM_S 为 4KB

  2. 修改链接脚本参数（路径：{SDK}\amebaxxx_gcc_project\amebaxxx_layout.ld）：

    • 调整 KM4_IMG1_SIZE 参数值以修改 KM4_BOOTLOADER_RAM_S

  3. 重新编译工程生成新版Bootloader

  4. 若Bootloader过大，需同步修改 km4_boot_all.bin 结束地址并烧录：

备注

新Bootloader版本号需高于当前版本才能被 boot ROM 加载

调整堆内存(Heap)大小

系统堆内存由多区块组成，通过 component\os\freertos\freertos_heap5_config.c 中的 os_heap_init() 函数初始化。

默认PSRAM扩展配置

  • PSRAM_HEAP1_START 为无效地址

  • PSRAM_HEAP1_SIZE 为 0

初始堆内存定义
KM4_PSRAM_HEAP_EXT (rwx) : ORIGIN = 0x6FFFFFFF, LENGTH = 0x6FFFFFFF - 0x6FFFFFFF  /* KM4 PSRAM 扩展堆 */
KM0_PSRAM_HEAP_EXT (rwx) : ORIGIN = 0x6FFFFFFF, LENGTH = 0x6FFFFFFF - 0x6FFFFFFF  /* KM0 PSRAM 扩展堆 */

KM4 堆扩展

当 KM4 堆内存不足时，按以下步骤扩展：

  1. 在链接脚本 amebaDplus_layout.ld 中定义未使用的 PSRAM 区域：

    • 设置有效的 Heap Start 起始地址

    • 配置 Heap Size 值

  2. 使用系统接口添加堆内存区块：

    bool os_heap_add(u8 *start_addr, size_t heap_size);

  3. 重新编译并烧录固件使配置生效

备注

链接脚本(amebaDplus_layout.ld)中定义的符号需在 ameba_boot.h 中声明后方可使用

KM0 堆扩展

当 KM0 堆内存不足时，修改 KM0_PSRAM_HEAP_EXT 参数配置即可

默认PSRAM扩展配置

  • PSRAM_HEAP1_START 为无效地址

  • PSRAM_HEAP1_SIZE 容量为 0

初始堆内存定义
KM4_PSRAM_HEAP_EXT (rwx) : ORIGIN = 0x6FFFFFFF, LENGTH = 0x6FFFFFFF - 0x6FFFFFFF  /* KM4 PSRAM 扩展堆 */
KR4_PSRAM_HEAP_EXT (rwx) : ORIGIN = 0x6FFFFFFF, LENGTH = 0x6FFFFFFF - 0x6FFFFFFF  /* KR4 PSRAM 扩展堆 */

KM4 堆扩展步骤

当 KM4 堆内存不足时，按以下流程扩展：

  1. 在链接脚本 amebalite_layout.ld 中定义未使用的 PSRAM 区域：

    • 设置有效的 Heap Start 起始地址

    • 配置 Heap Size 值

  2. 调用系统接口添加堆内存区块：

    bool os_heap_add(u8 *start_addr, size_t heap_size);

    备注

    地址必须指向有效的 PSRAM 空间

  3. 重新编译工程并烧录新固件激活扩展堆

  • PSRAM_HEAP1_SIZE 容量为 0

初始堆内存定义
KM4_PSRAM_HEAP_EXT (rwx) : ORIGIN = 0x6FFFFFFF, LENGTH = 0x6FFFFFFF - 0x6FFFFFFF  /* KM4 PSRAM 扩展堆 */
KR4_PSRAM_HEAP_EXT (rwx) : ORIGIN = 0x6FFFFFFF, LENGTH = 0x6FFFFFFF - 0x6FFFFFFF  /* KR4 PSRAM 扩展堆 */

KM4 堆扩展步骤

当 KM4 堆内存不足时，按以下流程扩展：

  1. 在链接脚本 amebalite_layout.ld 中定义未使用的 PSRAM 区域：

    • 设置有效的 Heap Start 起始地址

    • 配置 Heap Size 值

  2. 调用系统接口添加堆内存区块：

    bool os_heap_add(u8 *start_addr, size_t heap_size);

    备注

    地址必须指向有效的 PSRAM 空间

  3. 重新编译工程并烧录新固件激活扩展堆

备注

  • 链接脚本 (amebalite_layout.ld) 中定义的符号需在 ameba_boot.h 中声明

  • KR4 堆内存扩展方法与此流程相同
