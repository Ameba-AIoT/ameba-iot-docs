备注

  • SPI DEV_TX_REQ：Ameba 通过此引脚上升沿通知 Host 有数据待发送

  • SPI DEV_READY：Ameba 状态指示信号

  • Host 端 SDIO 中断模式要求：

    • SDIO_DATA1 必须配置为 SDIO 功能（非GPIO）。

    • 若 Host 不支持 SDIO 中断，可切换为轮询模式。

Wi-Fi Bridge 移植指南

Device 驱动移植

  1. 工程配置

    1. 在工程目录执行配置命令（示例路径： {SDK}/amebaxxx_gcc_project)

      ./menuconfig.py

    2. 依次进入 CONFIG WHC ITNF > WHC Mode > WHC_BRIDGE > HW INTERFACE，选择桥接模式和需要的接口

      选择 WHC_INTF_SDIO：

      (Top) -> CONFIG WHC INTF
----Configuration----
    WHC MODE (WHC_BRIDGE) --->
    HW INTERFACE --->
        (X) WHC_INTF_SDIO
        ( ) WHC_INTF_USB
        ( ) WHC_INTF_SPI

      选择 WHC_INTF_SPI：

      (Top) -> CONFIG WHC INTF
----Configuration----
    WHC MODE (WHC_BRIDGE) --->
    HW INTERFACE --->
        ( ) WHC_INTF_SDIO
        ( ) WHC_INTF_USB
        (X) WHC_INTF_SPI

  2. 固件编译

    执行编译命令生成固件：

    ./build.py

  3. 固件烧录

    使用烧录工具将生成的固件写入开发板。

Host 驱动移植

当前驱动代码已在 Linux Kernel v5.15 完成验证，若在其他内核版本遇到编译问题，请联系 <claire_wang@realsil.com.cn>。

本地SDK环境操作

  1. 进入驱动配置目录

    cd {SDK}/component/wifi/whc_host_linux

  2. 执行接口部署脚本（支持 SDIO/SPI 等接口）

    ./fullmac_setup.sh sdio bridge

    ./fullmac_setup.sh spi bridge

    备注

    SPI 会在 CS 引脚置低并检测到 CLK 后立即进行数据传输。总线繁忙时可能存在边界条件：

    • 测试表明，在 CS 置低与主机推送 CLK 之间增加 7μs 延迟可确保总线繁忙时的数据传输安全。

    • 但部分旧版本系统的 SPI 驱动可能不支持 spi_delay，此时需直接修改驱动代码以添加延迟。

  3. 在交互界面中选择目标设备型号

  4. 将 whc_host_linux 目录完整拷贝至Linux 主机

Linux主机操作

  1. 安装编译工具链

    sudo apt-get install build-essential

  2. 编译并加载驱动模块

    cd {driver_path}/whc_host_linux
make
sudo cp bridge/sdio_bridge/bridge_sdio.ko /lib/modules/$(uname -r)/
sudo systemctl stop dhcpcd.service
sudo depmod -a
sudo modprobe bridge_sdio

    cd {driver_path}/whc_host_linux
make
sudo cp bridge/spi_bridge/bridge_spi.ko /lib/modules/$(uname -r)/
sudo systemctl stop dhcpcd.service
sudo depmod -a
sudo modprobe bridge_spi

    备注

    • {driver_path}：驱动目录实际存储路径

    • $(uname -r)：自动获取当前内核版本

  3. 启用网络接口

    sudo ifconfig eth_sta0 up
sudo ifconfig eth_sta1 up

  4. 验证接口状态

    ifconfig

当前驱动代码已在 Raspberry Pi 4 Model B （Broadcom BCM2711）和 Raspberry 5 （Broadcom BCM2712）完成硬件验证，它们分别兼容Raspberry Pi OS 基于 Debian 11 和 Debian 12 的发行版。

  1. 启用 SDIO 接口配置

    • 树莓派4或者4B：

      # Add device tree overlay in /boot/config.txt
sudo sh -c 'echo "dtoverlay=sdio,poll_once=off" >> /boot/config.txt'

    • 树莓派5：

      # Add device tree overlay in /boot/firmware/config.txt
sudo sh -c 'echo "dtoverlay=sdio,poll_once=off" >> /boot/firmware/config.txt'

    备注

    此配置将禁用 SDIO 总线轮询机制以提升传输稳定性。

  2. 后续操作与 Linux 环境 步骤 3~7 完全兼容

  1. 在树莓派中使能SPI接口

    sudo raspi-config

  2. 选择 Interface Options > SPI > Yes

  3. 生成设备树 overlay

    sudo su
cd {driver_path}/whc_host_linux/common/spi
dtc -@ -Hepapr -I dts -O dtb -o inic_spidev.dtbo spidev-overlay.dts
cp inic_spidev.dtbo /boot/overlays/
dtoverlay inic_spidev

  4. 后续操作与 Linux 环境 步骤完全兼容

whc_bridge_host_buf_rx_to_user

__attribute__((weak)) int whc_bridge_host_buf_rx_to_user(u8 *buf, u16 size);

项目

描述

简介

接收从设备到主机的数据。 这是一个弱函数，SDK 会将数据从内核发送到用户空间。

不使用 Bridge APP 的客户需要自己实现该函数。

参数

  • buf：接收到的数据地址

  • size：接收到的数据长度（包含填充）

返回值

数据接收的结果

  • 0：数据正常传送到用户空间

  • 其他：数据接收异常

备注

  • 在我们提供的内核驱动中，网络设备的物理地址（MAC 地址）默认被设置为 00:00:00:00:00:00。在开发应用程序时，客户需要使用 setmac 命令为网络设备设置正确的物理地址。有关此过程的详细信息，请参考 getmac 和 setmac 命令。

Wi-Fi Bridge MP 介绍

在Wi-Fi Bridge上面实现 MP（Mass Production）流程有如下两种方式：

  • 如果设备端（Ameba）的调试串口（LOGUART）是引出的，可以直接通过设备端的调试串口去下 MP 命令以完成 MP 测试。

  • 如果设备端的调试串口没有引出, 客户需要基于我们提供的接口函数实现一条在主控端到设备端的 MP 命令的透传通路。

  • 这种方式几乎和标准的 FreeRTOS MP 流程一样，可以参考章节 MP 测试。

  • 为了减少来自主控端的干扰，最好调用 whc_bridge_dev_api_set_host_state(WHC_BRIDGE_HOST_UNREADY) 将Wi-Fi Bridge 主控端设置为非就绪状态。

在 Wi-Fi Bridge API 中定义了主控端应用层和设备端之间通信的接口函数。这一系列接口函数可以帮助客户实现主控端应用层与设备端之间的通信系统，其通信流程如下：

  1. 在主控端，系统向设备端发送 MP 命令。

  2. 在设备端，系统调用 wext_private_command() 对接收到的命令进行解析和执行。

    命令的响应或处理结果将存储在 wext_private_command() 的关键输入参数 user_buf 中。

  3. 在设备端，系统将 user_buf 中的内容传回到主控端，并由主控端的系统显示接收到的信息。

关于 MP 命令的详细信息，请参阅 MP 测试。
