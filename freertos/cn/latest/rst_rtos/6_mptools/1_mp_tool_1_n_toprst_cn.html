概述

本章节介绍如何使用 Ameba 1-N MP ImageTool 同时烧录N个模块。

环境配置

硬件环境

如果需要同时烧录N个终端模块，需要具备如下条件：

  • 终端设备（Ameba终端需要有 LOGUART 接口以及 UART_DOWNLOAD 按键）

  • USB 集线器和 micro-USB 连接器

  • PC 和 Ameba 1-N MP ImageTool烧录工具

备注

如果借助外部 UART 工具烧录，则还需要一个 USB-UART 适配器。

软件环境

  • 环境要求：如WinXP，Win7或更高版本，Microsoft .NET Framework4.0。

  • 软件位置：

    • Ameba_1-N_MP_ImageTool.exe：{SDK}/tools/ameba/MP_ImageTool_1_N/Ameba_1-N_MP_ImageTool.exe

    • Device Profiles Editor：{SDK}/tools/ameba/DeviceProfileEditor/AmebaDeviceProfileEditor.exe

    • Device Profiles：{SDK}/tools/ameba/ImageTool/Devices/Profiles

备注

  • MP ImageTool 将在后面章节中指代 {SDK}/tools/ameba/MP_ImageTool_1_N/Ameba_1-N_MP_ImageTool.exe。

  • 如需通过UART接口下载固件，需要先安装板载USB转UART适配器（例如PL2303GC）的主控制器驱动，具体驱动请在相应USB转UART适配器厂商的官方网站查询。

  • 禁止修改或删除 floader_amebaxxx.bin 文件, 因为烧录工具会先把 Flashloader 烧录进 SRAM，然后 Flashloader 负责后续量产固件烧录到Ameba Flash。

  • 对于WinXP或Win7，若需通过USB接口下载固件，需要先安装USB驱动 {SDK}/tools/ameba/MP_ImageTool_1_N/RtkUsbCdcAcmSetup.INF。

固件下载

开始下载固件之前，请确保 DUT 进入 UART_DOWNLOAD 模式，之后可以开始下载固件。

根据 DUT 所处的状态，进入 UART_DOWNLOAD 模式的操作方法如下：

空片状态可以直接开始烧录，MP ImageTool会设置空片模块进 UART_DOWNLOAD 模式。

系统正常启动，可以接收AT指令状态，MP ImageTool会设置DUT进 UART_DOWNLOAD 模式。

如果模组系统启动异常，请通过如下方式进入 UART_DOWNLOAD 模式。

  1. LOGUART Tx 引脚接GND。

  2. 重新接通设备电源。

  3. 断开 LOGUART Tx 引脚和GND。

  • 如果公版处于系统正常可以接收AT指令状态，无需额外操作，可以直接下载固件。

  • 如果公版处于异常状态，可通过如下方式进入 UART_DOWNLOAD 模式。

    1. 按下 UART_DOWNLOAD 按钮并保持按下状态。

    2. 重新接通设备电源或按下 CHIP_EN 按钮。

    3. 松开 UART_DOWNLOAD 按钮。

MP ImageTool 支持图形界面下载和命令行下载两种模式，以下进行详细介绍。

图形交互模式

下载步骤

图形界面固件下载步骤如下：

  1. 双击 Ameba_1-N_MP_ImageTool.exe。

  2. 点击 Chip Select 选择芯片型号。

  3. 配置参数：

    Baudrate:

    固件烧录波特率，默认值是1500000

    Image:

    点击 Browse 按键选择待烧录的目标固件

    Addr(hex):

    烧录固件的起始地址

    Leg(hex):

    指定固件烧录长度，一般选择目标固件的时候会自动计算该值

备注

一旦芯片型号选定，界面上灰掉的按键/复选框表示该功能不支持当前芯片型号。

  1. 点击 Scan Device 按键, 显示框会列出当前PC连接的DUT端口

    • 如果使能 USB Download ， 扫到的可用端口为 USB COMs.

    • 如果不使能 USB Download ，扫到的可用端口为 LOGUART COMs.

    备注

    如果没有正确显示可用端口，可以检查设备驱动是否没有安装或者正在安装，这种情况一般在DUT第一次接入PC的时候会出现。

  2. 选择要下载固件的DUT端口.

  3. 确认DUT是空片或者已进入 UART_DOWNLOAD 模式或者在正常可以接收 AT 指令状态

  4. 点击 Download ，开始下载固件

如下图所示，固件下载过程中，每个DUT端口后面会显示一个绿色进度条，表示当前DUT下载进度。进度条前端有计时显示，后端有一个小灯泡，鼠标放在小灯泡上，可以指示当前烧录状态。烧录结束，小灯泡变成绿色对勾，表示正常烧录结束。

如果下载过程遇到问题，可以参考 常见问题 章节，确认是否为常见典型问题。

附加功能

工厂模式

使能工厂模式，图形界面上某些功能会被禁掉，工厂模式的配置参数会保存在 Setting.ini，所以下次重启也会继续开启工厂模式。

如果需要退出工厂模式，有两种方式可选：

  1. 关闭Ameba 1-N MP ImageTool.exe.

    1. 删掉 Setting.ini.

    2. 修改 Setting.ini 中 FactoryMode 参数为 False.

  2. 重新打开Ameba 1-N MP ImageTool.exe，图形界面就会恢复正常可操作模式。

校验

此功能是指下载结束，是否需要做校验，确认下载进DUT的固件正确。

方法：在 下载步骤 第三步时勾选 Verification 复选框。

芯片擦除

该功能表示是否要在下载开始之前对Flash全片擦除。

方法：在 下载步骤 第三步时勾选 ChipErase 复选框。

跳过系统

此功能表示下载过程是否跳过系统数据区域。

方法：在 下载步骤 第三步时勾选 Skip System 复选框。

USB 下载

此选项表示是否通过USB端口下载固件。

方法：在 下载步骤 第三步时勾选 USB Download 复选框。

如果使能USB下载，需要点Profile Browse 键导入目标芯片的profile文件。

跳过 Wi-Fi校准

此选项表示下载过程是否跳过 Wi-Fi校准。

方法：在 下载步骤 第三步时勾选 Skip Wifi calibration 复选框。

支持大于16M Flash下载

该功能表示是否支持大于16MB Flash的下载，如果使能，功能会帮助使用者编辑OTP位，以使能芯片4-byte Flash寻址功能。

方法：在 下载步骤 第三步时，勾选 Option –> Support for >16MB Flash Download 。

保存擦除/下载过程的Flash写保护处理方法

下载固件过程中，可能会遇到Flash写保护置位，导致下载不了的情况，此时MP ImageTool会弹窗，引导使用者做进一步操作处理，该功能的作用就是保存当前的处理方式，下一次遇到Flash写保护置位的情况，会直接按第一次操作方法处理，无需再次人为介入。

方法：在 下载步骤 第三步时，勾选 Option –> Remember Flash Protection process for erase/download 。

命令行模式

下载步骤

MP ImageTool同时也支持命令行模式下载。在Windows的命令行窗口执行 Ameba_1-N_MP_ImageTool.exe 并且带着指定的参数，v2.5.11版本之后，命令行模式支持一次带入多参数指令模式。

命令行下载固件方法如下：

  1. 必须设置参数项

    1. 芯片型号

      参数 -c/--chip [amebasmart|amebalite|amebadplus] 带入目标IC型号。

      $Ameba_1-N_MP_ImageTool.exe -c amebaxxx
CHIP:            AMEBAXXX

    2. 波特率

      参数 -b/--baudrate <value> 带入下载波特率参数.

      $Ameba_1-N_MP_ImageTool.exe -b 1500000
BAUDRATE:            1500000

      备注

      波特率可设置参数为 115200, 128000, 153600, 230400, 380400, 460800, 500000, 921600, 1000000, 1382400, 1444400, 和 1500000。

    3. 固件选择

      参数 -i/--image <image_path> 带入要下载的固件。

      $Ameba_1-N_MP_ImageTool.exe -i E:\test\image_path\image_all.bin
FILE PATH:            e:\test\image_path\image_all.bin

    4. 下载起始地址

      参数 -a/--address <value> 带入固件要下载的起始地址。

      $Ameba_1-N_MP_ImageTool.exe -a 0x08000000
ADDRESS:            0x08000000

    5. 固件大小

      参数 -l/--length <value> 带入要下载的固件的大小，如果下载默认固件长度，可以不计算，此处带入0x0即可，工具会自动计算固件长度。

      $Ameba_1-N_MP_ImageTool.exe -l 0xa500
LENGTH:  0XA500

      备注

      此参数一定要带入要下载的固件长度或者0x0，如果不带入，工具会使用Setting.ini文件的默认值，有可能不符合实际固件大小。

  2. 检查设置参数

    参数 --show setting 可以列出当前设置的参数情况。

    $Ameba_1-N_MP_ImageTool.exe --show setting
CHIP:            AMEBAZ
BAUDRATE:       1500000
IMAGE:           e:\test\image_path\image_all.bin
ADDRESS:        0x08000000
SKIP SYSTEM:     False
LENGTH:          0x0
ProcessFlashBPsForErase:    0
ProcessFlashBPsForDownload:    0
VERIFICATION:    True
SAVELOG:         True

    备注

    下载开始前，这些参数只需要设置一次，MP ImageTool会保存在根目录的Setting.ini文件。

  3. 固件下载

    配置好参数之后，参数 -d/--download <dev1> <dev2> 或 -d/--download added 带入目标端口，即可开始下载。

    • -d/--download added 可以给所有通过 --add device 指令添加过的端口下载固件。

      $Ameba_1-N_MP_ImageTool.exe -d added

    • -d/--download <dev1> 可以下载指定的<dev1>， 并且不会修改通过 --add device 指令添加的端口列表。

      $Ameba_1-N_MP_ImageTool.exe -d COM4 COM5
1  COM4 SUCCESS
2  COM5 SUCCESS

      备注

      v2.5.11版本之后, 命令行参数支持一次带入多个参数执行下载。

      $Ameba_1-N_MP_ImageTool.exe -c amebad -b 1500000 -a 0x08000000 -l 0xa500 -i e:\test\image_path\image_all.bin -s true -d COM4 COM5
1  COM4 SUCCESS
2  COM5 SUCCESS

下载结束，如果打印信息显示下载失败，可以查看根目录 log 文件夹下的日志文件，查找失败原因。

附加功能

USB下载

如果目标IC支持USB下载，在 下载步骤 第1步设置。

方法：参数 -u/--usbdownload [true|false] 设置是否使能该参数.

$Ameba_1-N_MP_ImageTool.exe –u true
USBDOWNLOAD:    True

如果使能USB下载，需要同时通过参数 -p/--profile <path> 带入profile。

$Ameba_1-N_MP_ImageTool.exe –p E:\MP_Image_Tool\image_all.rdev
PROFILE:  e:\mp_image_Tool\image_all.rdev

扫描DUT

参数 --scan device 可以扫描当前连接到PC的串口设备。

$Ameba_1-N_MP_ImageTool.exe --scan device
USB Serial Port (COM4)
USB Serial Port (COM5)

添加/移除DUT端口

参数 --add <dev> / --remove <dev> 表示添加/移除需要下载的DUT端口， --show device 可以列出当前添加的DUT端口。

$Ameba_1-N_MP_ImageTool.exe --add COM4 COM5
COM4 added!
COM5 added!
$Ameba_1-N_MP_ImageTool.exe --show device
COM4
COM5
$Ameba_1-N_MP_ImageTool.exe --remove COM5
COM5 removed!
$Ameba_1-N_MP_ImageTool.exe --show device
COM4

如果DUT端口已添加到下载列表，固件下载方式可以参考 下载步骤 第4步。

校验

参数 -v/--verify [true|false] 表示下载结束之后是否要做校验。

$Ameba_1-N_MP_ImageTool.exe –v true
VERIFICATION:  True

该参数需要在 下载步骤 第1步带入。

全片擦

参数 -e/--chiperase [true|false] 表示在下载固件之前是否要对Flash做全片擦除。

$Ameba_1-N_MP_ImageTool.exe –e true
CHIPERASE:  True

该参数需要在 下载步骤 第1步带入。

DTR/RTS配置

如果设备支持DTR/RTS控制模块自动进 UART_DOWNLOAD 模式，参数 -g/--dtrrtsconfig [true|false] 可以使能/不使能该功能.

$Ameba_1-N_MP_ImageTool.exe –g true
DtrRtsConfig:  True

该参数需要在 下载步骤 第1步带入。

保存擦除/下载过程的Flash写保护处理方法

下载固件过程中，可能会遇到Flash写保护置位，导致下载不了的情况，此时下载工具会弹窗，引导使用者做进一步操作处理，该功能的作用就是保存当前的处理方式，下一次遇到Flash写保护置位的情况，会直接按第一次操作方法处理，无需再次人为介入。 参数 -f/--processflashbpsforerase | -n/--processflashbpsfordownload <value> 可以帮助设置是否使能该功能。

$Ameba_1-N_MP_ImageTool.exe –f 1
ProcessFlashForErase:  1
$Ameba_1-N_MP_ImageTool.exe –n 1
ProcessFlashForDownload:  1

该参数需要在 下载步骤 第1步带入。

保存日志

参数 -s [true|false] 可以设置是否保存下载日志。

$Ameba_1-N_MP_ImageTool.exe –s true
SAVELOG:    True

该参数需要在 下载步骤 第1步带入。

GPIO/PWM 指示

MP ImageTool支持通过USB下载固件，由于DUT和USB端口没有唯一的对应关系，因此在生产线上无法快速识别下载失败的设备。如果需要通过USB下载固件，可以使用GPIO/PWM指示功能来指示哪些DUT已成功下载固件，以便为失败的DUT重新下载固件。

GPIO/PWM指示功能，表示下载结束后，MP ImageTool向DUT发送相应的指令，DUT会根据指令，向指定的GPIO输出高电平或低电平或通过PWM通道输出PWM。

具体的配置方法如下：

  1. 首先关闭Ameba 1-N MP ImageTool.exe

  2. 打开根目录 Setting.ini 文件并配置 ProgramConfig (默认值: 0).

    具体的 ProgramConfig 配置位描述，参考下表。

    Bit field description of ProgramConfig

    Bit

    Description

    [63:32]

    Download indication configuration:

    • For GPIO indication

      • Bit[63:33]: Reserved

      • Bit[32]: GPIO output level

    • For PWM indication

      • Bit[63:57]: PWM duty cycle (0~100, unit: percent)

      • Bit[56:32]: PWM period (unit: us)

    [31:30]

    Download indication strategy:

    • 0: Disable

    • 1: GPIO indication after download success

    • 2: PWM indication after download success

    • 3: Reserved

    [29:16]

    Download indication PIN name, refer to the SoC-specific definition of PinName for MBED API

    • GPIO: all GPIO pins can be selected

    • PWM: PWM channels can be selected

    [15:2]

    Reserved

    [1:0]

    NAND Flash bit flip fail level:

    • 2/3: Reserved

    • 1: Fail at fatal bit flip error, i.e. bit flip count > ECC level

    • 0: Fail at bit flip error, i.e. bit flip count >= ECC level

  3. 保存 Setting.ini 重新打开工具开始下载.

读MAC地址

指令 -r/--readmac 可以协助读取设备的MAC。

$Ameba_1-N_MP_ImageTool.exe -r COM4
COM4  FF-FF-FF-FF-FF-FF

合并固件

参数 --combine "<image1_path>;<image2_path>" --offset "<image1_offset><image2_offset>" [--output "<image_all_output>"] 可以协助用户合并固件。

$Ameba_1-N_MP_ImageTool.exe --combine "E:\image1.bin;E:\image2.bin" --offset "0x00000000;0x00004000" --output E:\image_all.bin
Offset0: 0x00000000, Image0: E:\image1.bin
Offset1: 0x00004000, Image1: E:\image2.bin
Images have been combined successfully, size: 12KB, path: E:\image_all.bin

擦除

参数 --erase <address> <value> [length] <value>(KB) <dev1> <dev2> 用于命令行模式擦除Flash。

$Ameba_1-N_MP_ImageTool.exe --erase 0x08000000 1024 COM56
Flash erase done!

备注

擦Flash之前，需要确认相关参数已经设置就绪，例如：固件、波特率等。

通用配置

通用设置一般直接以参数形式放在 Setting.ini 文件中，可以通过编辑该文件实现配置。

备注

在配置 Setting.ini 文件之前，一定要先关掉烧录工具。

显示详细日志

此项配置用于显示烧录过程的详细日志，方便用户查看烧录过程遇到的问题，参数为 Setting.ini 文件中的 ShowDetailLog：

  • True：启用

  • False：禁用

ShowDetailLog           =True

该参数需要在烧录开始之前设置。

下载进度条显示

Setting.ini 文件的 DownloadProgressDisplay 用于配置是否在命令行模式下载时，在控制台显示下载进度。

DownloadProgressDisplay           =True

该参数也可以在 下载步骤 第1步带入。

常见问题

下载失败

如果固件下载失败，试着强制进 UART_DOWNLOAD 模式再次下载。

如果传输过程被中断，或者校验失败，进度条最后会有一个红底叹号，可以查看日志，试着解决导致失败的问题，重新下载。查看日志的方式有两种：

  • 把鼠标移动到红底叹号上，错误信息即显示出来，如下图示。

  • 如果使能保存日志功能，检查根目录 log 文件夹的日志文件，获取详细的下载日志，查看失败原因。

    Start Tool from UI
Scan Device Start
Scan Device End
Start to Download...
COM56 Open COM error

备注

这种查看方式，需要确保下载开始之前，使能保存日志功能。

Flash 写保护

如果Flash被写保护了，即在下载或擦除Flash的时候，工具检测到Flash写保护被置位，会弹窗允许用户决定下载工具接下来的操作。

  • 如果有需要，可以允许工具修改Flash的写保护，以完成下载或擦除操作，如下图所示。

  • 或者，可以在 Setting.ini 文件设置 ProcessFlashBPsForErase 和 ProcessFlashBPsForDownload 保存，以防产线在量产过程中出现误选导致下载失败。

    参数 ProcessFlashBPsForErase 和 ProcessFlashBPsForDownload 表示：

    • 0：When detect Flash block protection, tool pops new window to ask the customer what to do next when erase/download.

    • 1：Tool will remove the Flash protection and restore the Flash protection when erase/download.

    • 2：Tool will abort erase/download when detect Flash block protection.

下载过程中断电

如果在下载过程中偶发终端掉电问题，会导致下载失败，这时可以重新进 UART_DOWNLOAD 模式并重新开始下载。
